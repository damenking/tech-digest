Yann LeCun's AMI Labs raises $1.03 billion to build AI that understands the physical world
Yann LeCun, the Turing Prize-winning AI researcher who departed Meta earlier this year, has closed a $1.03 billion funding round for his new startup AMI Labs at a $3.5 billion pre-money valuation. The company's focus is on building 'world models' — AI systems that understand physical reality and causality rather than just predicting the next token in a sequence. LeCun has been one of the most vocal and persistent critics of the large language model paradigm, arguing for years that LLMs are fundamentally limited because they don't ground language in physical understanding. His view is that human-level AI has to come from mastering the physical world, not from scaling language models further. AMI Labs appears to be his attempt to prove that thesis in practice rather than just in conference talks and Twitter debates. The $3.5 billion pre-money valuation is remarkable given the company is brand new and has no products yet — a sign of how much weight LeCun's name and track record carry in the investment community. For context, LeCun spent over a decade at Meta AI (FAIR) and was instrumental in making deep learning mainstream. The funding also signals that investors aren't done betting on alternative AI architectures even as GPT-5.4 and Gemini 3 dominate the current moment. This is one to watch closely — if anyone has the credibility and resources to mount a serious challenge to the transformer-centric status quo, it's LeCun. Whether world models can actually deliver on the promise in a reasonable timeframe is the open question.

Building a procedural hex map with Wave Function Collapse — the HN community's favorite post today
A detailed technical writeup on building procedurally generated hex maps using the Wave Function Collapse (WFC) algorithm has shot to the top of Hacker News with over 400 points, making it the most engaged technical post of the day. WFC is a constraint propagation algorithm that was popularized by Maxim Gumin around 2016 and has become a go-to tool for procedural content generation in games and simulations. The algorithm works by taking example patterns and generating new content that locally matches the statistics of those examples, which makes it excellent for generating tile-based maps that look coherent without hand-crafting every cell. Hex grids add an interesting dimensional challenge compared to square grids because each cell has six neighbors rather than four, which changes the constraint propagation topology. The writeup apparently goes into enough implementation detail to be genuinely educational, which is why it's resonating with the HN crowd — procedural generation is one of those topics with a passionate community of both game developers and generalist programmers. For developers working on games, simulations, or any domain where you need large amounts of structured but varied content, WFC is worth understanding deeply. JSLinux now supports x86_64 — full 64-bit Linux in the browser, no install required
Fabrice Bellard's JSLinux project, which runs a full Linux environment inside the browser using JavaScript, has been updated to support x86_64 — the 64-bit architecture that essentially all modern software is compiled for. Bellard is something of a legend in systems programming — he's the creator of QEMU, FFmpeg, and tcc (the Tiny C Compiler), among other landmark projects, so this isn't a hobbyist experiment but a serious technical achievement from someone who knows this problem space deeply. For developers, educators, and anyone who wants to give someone a Linux environment without any setup friction, JSLinux is now dramatically more useful. The performance won't match native execution, but for many use cases — running scripts, exploring the filesystem, teaching Linux concepts — it's more than adequate. Is AI code relicensing via 'clean room' implementation legal? The copyleft erosion question
A thoughtful essay on Hacker News (nearly 400 points) tackles a genuinely thorny legal and ethical question: can AI coding agents effectively launder GPL-licensed code by creating a 'clean room' reimplementation, stripping the copyleft requirement in the process? The clean room technique is a legally recognized method dating back to the 1980s — Compaq used it to clone the IBM BIOS — but it requires a strict process where one team reverse-engineers the specification and another team, with no access to the original, implements against that spec. The question is whether AI agents, which have been trained on vast amounts of GPL and copyleft code, can legitimately perform both halves of this process in a way that produces legally clean output. Simon Willison raised a related version of this question specifically about the chardet library, noting that coding agents are remarkably good at producing clean-room-style implementations. If AI can effectively relicense copyleft code at scale, it would be a significant blow to the copyleft software model that underpins projects like Linux, GCC, and thousands of others. The legal analysis here is genuinely unsettled — courts haven't ruled on AI-generated clean room implementations, and the 'intent' question that courts sometimes apply to clean room claims is murky when one party is an AI. For open source maintainers and lawyers, this is a live and important question. For engineers, it's worth understanding the risk profile before using AI to reimplement something you know is GPL-licensed.
