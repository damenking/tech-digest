GLM-5.2: Chinese Lab Z.ai Releases 753B Open-Weights Model Under MIT License
Z.ai, a Chinese AI lab, has released GLM-5.2 as open weights under an MIT license — and the early read from Simon Willison and Hacker News is that it may be the most capable text-only open-weights LLM available right now. It was first released to coding plan subscribers on June 13th before going fully open on June 16th. The MIT license is notably permissive, meaning commercial use is unrestricted — a meaningful choice that signals Z.ai wants broad adoption over control. Hacker News has a dedicated comparison thread (GLM-5.2 vs. Opus) with a score of 122, suggesting practitioners are already running head-to-head evaluations against Anthropic's Claude Opus. For developers and researchers who want frontier-level text capabilities without API costs or vendor lock-in, this is a genuinely big deal. The MoE architecture means you don't need to run all 753B parameters simultaneously, which makes inference more feasible on high-end consumer or prosumer hardware — though 1.51TB still demands serious infrastructure. The text-only limitation is worth noting: no vision or multimodal capability, which rules it out for some use cases. The broader pattern here is that the open-weights frontier keeps moving up, compressing the advantage that closed labs have been able to maintain.

Microsoft Discovers Lightweight Cryptocurrency-Stealing Backdoor Spreading via USB
Microsoft has identified a new piece of malware — dubbed 'Crypto Clipper' — that spreads via USB drives and uses Tor for command-and-control communications, making it a notably stealthy threat. Crypto Clippers are a well-established malware category that intercept clipboard contents to swap cryptocurrency wallet addresses, redirecting transactions to attacker-controlled wallets; the victim types their wallet address, the malware silently substitutes a different one, and the funds go to the attacker. What makes this variant interesting is the USB propagation vector, which means it can jump air-gapped or network-isolated systems — a technique associated with more sophisticated threat actors. Tor-based C2 makes network-level detection significantly harder since the traffic blends with legitimate Tor usage. The lightweight nature of the backdoor suggests it's designed for persistence and stealth rather than large-scale data exfiltration. For organizations with cryptocurrency holdings or employees who manage crypto wallets — increasingly common in fintech and Web3 companies — this is a direct operational risk. USB-propagating malware has a long history, from Stuxnet to various APT campaigns, and the vector remains effective because people still use USB drives, especially in contexts where file sharing over network is restricted. The standard mitigations apply: disable autorun, restrict USB access on sensitive systems, use hardware wallets with on-device address verification.

sqlite-utils 4.0rc1 Adds Migrations and Nested Transactions
Simon Willison has released sqlite-utils 4.0rc1, a release candidate for the major version update to his widely-used Python library and CLI tool for working with SQLite databases. The two headlining features are migrations and nested transactions — both significant additions that address longstanding gaps in the library's functionality. Migrations are essential for any project where the database schema evolves over time, which is basically any real project; previously, sqlite-utils was excellent for ad hoc data work but required external tools or manual schema management for production-grade use. Nested transactions (implemented via SQLite's savepoint mechanism) allow for more nuanced transactional control, where you can roll back part of a transaction without losing everything — particularly useful in complex data pipelines. Willison's sqlite-utils has become a beloved tool in the Python data ecosystem, especially among people working with structured data, journalism datasets, and the broader Datasette ecosystem. The 4.0 version bump signals breaking changes from 3.x, so users should review the migration guide before upgrading production deployments. The combination of migrations and nested transactions pushes sqlite-utils closer to being a viable lightweight ORM alternative for applications that want SQLite's simplicity without the overhead of SQLAlchemy. For developers who've been using sqlite-utils for data exploration and want to graduate it into a more production role, this release makes that more practical.
