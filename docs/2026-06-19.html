GLM-5.2 is probably the most powerful text-only open-weights LLM right now
Chinese AI lab Z.ai dropped GLM-5.2 on June 16th under an MIT license, and it's a genuinely impressive release. The model is a 753-billion-parameter Mixture of Experts architecture with 40 billion active parameters, weighing in at 1.51TB — so running it locally is not for the faint of heart or light of wallet. It's text-only (no multimodal inputs), but in that lane it reportedly beats everything else currently available as open weights. Simon Willison flagged it as the most capable text-only open-weights model available today, which is a meaningful claim given how crowded the open-weights space has gotten. The MIT license is notable — it means commercial use is unrestricted, which is a deliberate move to encourage adoption and build an ecosystem around Z.ai. This follows a pattern of Chinese labs releasing extremely capable open-weights models to compete with the closed offerings from OpenAI and Anthropic. For engineers who want to self-host a frontier-class model for coding or reasoning tasks, this is worth evaluating seriously. The sheer size means you'll need serious infrastructure, but cloud-hosted inference via APIs is likely to follow quickly. The Mixture of Experts architecture means it's more efficient than a dense 753B model would be, but you still need to load the full parameter set into memory for serving. Worth keeping an eye on benchmarks as the community starts stress-testing it.

Google DeepMind's DiffusionGemma achieves 4x faster text generation
Google DeepMind released DiffusionGemma, a new model that delivers text generation roughly four times faster than comparable autoregressive approaches. The speed gain comes from applying diffusion-style generation — a technique that's been dominant in image generation for years — to text, which has historically been much harder to do well for language. Traditional language models generate text one token at a time in a left-to-right sequence, which creates an inherent latency floor. Diffusion models instead work by iteratively denoising a full output, which can be parallelized more aggressively. Previous attempts at diffusion-based language models struggled with quality, but the gap appears to be narrowing significantly. A 4x speed improvement at comparable quality would be a meaningful win for inference costs and latency-sensitive applications like real-time conversation. DeepMind hasn't published full benchmark comparisons yet, so independent evaluation will be important before drawing firm conclusions. If the quality-speed tradeoff holds up under scrutiny, this could be a significant architectural shift for production inference systems. It's also worth watching how this relates to Baseten and other inference infrastructure players — a fundamentally different generation architecture could require new optimization work up and down the stack.

Microsoft discovers new USB-spreading crypto-stealing backdoor that communicates over Tor
Microsoft's security team has identified a new piece of malware called 'Crypto Clipper' — a lightweight backdoor that spreads via USB drives and communicates with its command-and-control infrastructure over the Tor network. The USB propagation vector is a throwback to an older era of malware distribution, but it's clearly still effective, particularly in environments where network-based delivery is harder (air-gapped or semi-isolated systems). The Tor communication channel makes the malware significantly harder to detect and block at the network perimeter, since Tor traffic is encrypted and the exit nodes are constantly changing. The 'clipper' name refers to its primary payload: monitoring the clipboard for cryptocurrency wallet addresses and silently replacing them with attacker-controlled addresses, so any crypto transaction the victim initiates goes to the wrong place. This is a well-established attack pattern that has been surprisingly effective for years because users rarely double-check that the address they pasted is the one they intended. The combination of USB spread, Tor C2, and clipboard hijacking makes this a more sophisticated package than typical clippers. Organizations that handle cryptocurrency transactions should treat this as a meaningful threat and implement clipboard integrity checking and USB restrictions accordingly. The fact that it communicates over Tor also suggests the authors have some operational security sophistication.
