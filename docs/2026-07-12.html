Anthropic discovers a 'hidden space' where Claude puzzles through concepts before answering
Anthropic published research describing a new interpretability technique called the Jacobian lens, which gives them what they describe as the clearest view yet into what large language models are actually doing as they process a question. The tool surfaces what Anthropic researchers call a 'hidden space' — an intermediate representational layer where the model appears to work through concepts before generating output tokens. This is a significant advance in mechanistic interpretability, a field that has been trying for years to open the black box of how these models reason. The findings reportedly range from mundane (the model doing roughly what you'd expect) to genuinely unnerving (implications the researchers haven't fully characterized publicly yet). This matters because interpretability is one of the core unsolved problems in AI safety — if you can't understand what a model is doing internally, you can't reliably predict when it will fail or behave unexpectedly. Anthropic has been one of the more serious investors in interpretability research, and this feels like a real methodological step rather than a PR paper. The Jacobian lens approach is named after the mathematical object used to measure how changes in one layer affect outputs in another. For practitioners building on top of these models, this research is mostly background context for now, but it's the kind of foundational work that eventually informs how we audit and certify models for high-stakes deployments. Quantum error correction can now continuously recalibrate a quantum processor
Researchers have demonstrated a technique that uses quantum error correction data to continuously recalibrate a quantum processor in real time, rather than requiring separate calibration runs that take the system offline. The approach uses reinforcement learning to extract useful information from the error correction process itself — essentially turning the error signals into a feedback loop that adjusts the control algorithms on the fly. This is meaningful because quantum processors drift out of calibration constantly due to environmental noise, and the overhead of recalibration has been one of the practical bottlenecks to running longer quantum circuits reliably. By folding calibration into the error correction process that you're already running, you eliminate a category of interruption. The reinforcement learning angle is interesting — it's an example of classical ML being used to improve quantum hardware operations rather than the often-hyped idea of quantum computing improving ML. For anyone tracking the path toward fault-tolerant quantum computing, this is exactly the kind of systems-level engineering progress that matters more than raw qubit count announcements. It doesn't make quantum computers useful tomorrow, but it removes a real operational obstacle. The work was published with enough detail for Ars Technica to cover the mechanism substantively, which suggests it's a solid result rather than a preliminary demonstration.

Prefer strict tables in SQLite — a practical argument gaining traction
A blog post arguing for the use of SQLite's STRICT table mode scored 253 points on Hacker News this week, making it the highest-scoring community story in this digest. SQLite added STRICT tables as a feature to allow enforcing actual type constraints on columns — a significant departure from SQLite's traditional 'type affinity' system where you can store any type of data in any column. The argument for STRICT mode is essentially that SQLite's flexible typing is a footgun: it lets you insert a string into an integer column without error, which can cause subtle bugs that are hard to track down. For new projects, STRICT mode gives you the data integrity guarantees you'd expect from any other database. The high score suggests this resonates with a lot of developers who have either been bitten by SQLite's permissive typing or who appreciate having a clear best-practice recommendation. SQLite is increasingly used not just as a local database but as a primary production database for small-to-medium applications, embedded systems, and edge deployments — contexts where data integrity bugs can be especially painful. China's Tianwen-2 space probe has successfully rendezvoused with Kamo'oalewa, the near-Earth asteroid that orbits in a quasi-synchronous pattern with Earth — sometimes called Earth's 'quasi-moon.'
